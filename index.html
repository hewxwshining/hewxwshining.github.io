小程序 video 组件使用填坑记

video 组件

最近业务开发时遇到视频播放到结束前几秒的位置时自动停止播放的情况，而且在不同的播放环境下表现形式不一样：

  • 系统播放器环境下可以完整无卡顿播放；
  • google 浏览器中在结束前几秒自动停止，不会继续播放；

对于上面情况的视频，小程序 video 组件监听方法在开发者工具、Android和ios表现形式也不一样；

是否响应 bindplay bindpause bindended binderror bindtimeupdate
开发者工具 是 是 是 是 是
Android10 是 否 否 否 是
IOS 是 是 是 是 是

业务需求是需要在视频播放完后自动隐藏其自身，但是因为 android 无法监听到 bindend 与 bindpause 即使试了很多方法都没有用，最后只能监听到 bindtimeupdate 方法，

bindtimeupdate 播放进度变化时触发 ， 这个函数会返回一个当前播放进度， 跟视频的总时长，当当前进度大于总时长时隐藏自身，除了这个方法还没找到其他方法解决目前的问题

页面渲染，浏览器工作原理

1、 浏览器解析域名；

  • 浏览器缓存中没有该域名 IP 地址， 通过域名服务器（ NDS ）进行查找,返回该域名的IP地址；

  • 浏览器缓存中有该域名 IP 地址， 直接访问该IP地址；

2、 浏览器向服务器发起连接请求 （ TCP 三次握手）

3、 浏览器向服务器发起 HTTP GET 请求

对于网站来说，这个请求通常是一个HTML文件

3、服务器向浏览器相关的响应头和 THML 内容

  • 初始页面加载的14Kb规则，TCP慢开始就是初始响应为14Kb的原因

    TCP第一个响应包是14kb大小。这是慢开始的一部分，慢开始是一种均衡网络连接速度的算法。慢开始逐渐增加发送数据的数量直到达到网络的最大带宽。

4、浏览器解析返回的数据；

解析 是浏览器将通过网络接收的数据转换为 DOM 和 CSSOM 的步骤，通过渲染器把 DOM 和 CSSOM 在屏幕上绘制成页面

call、apply、bind、new 实现原理

Function.prototype.call()

call() 方法使用一个指定的 this 值和单独给出的一个或多个参数来调用一个函数。

该方法的语法和作用与 apply() 方法类似，只有一个区别，就是 call() 方法接受的是一个参数列表，而 apply() 方法接受的是一个包含多个参数的数组。

语法:

function.call(thisArg, arg1, arg2, …)

使用 call 方法调用父构造函数

使用 Food 构造函数创建的对象实例都会拥有在 Product 构造函数中添加的 name 属性和 price 属性,但 category 属性是在各自的构造函数中定义的。
