页面渲染，浏览器工作原理

1、 浏览器解析域名；

  • 浏览器缓存中没有该域名 IP 地址， 通过域名服务器（ NDS ）进行查找,返回该域名的IP地址；

  • 浏览器缓存中有该域名 IP 地址， 直接访问该IP地址；

2、 浏览器向服务器发起连接请求 （ TCP 三次握手）

3、 浏览器向服务器发起 HTTP GET 请求

对于网站来说，这个请求通常是一个HTML文件

3、服务器向浏览器相关的响应头和 THML 内容

  • 初始页面加载的14Kb规则，TCP慢开始就是初始响应为14Kb的原因

    TCP第一个响应包是14kb大小。这是慢开始的一部分，慢开始是一种均衡网络连接速度的算法。慢开始逐渐增加发送数据的数量直到达到网络的最大带宽。

4、浏览器解析返回的数据；

解析 是浏览器将通过网络接收的数据转换为 DOM 和 CSSOM 的步骤，通过渲染器把 DOM 和 CSSOM 在屏幕上绘制成页面

（1）处理 HTML 标记，并构建 DOM 树

DOM 树描述的HTML文档的内容。 是文档中的第一个标签，也是 DOM 树的根节点 。 DOM 反应了不同标记之间的关系和层次结构， 嵌套在其他标记中的标记为子节点， DOM 节点数量越多构建树做需要的时间也就也长；

当解析器发现非阻塞资源，例如图片、css文件时， 解析仍继续解析， 但是对于 <script> 标签（特别是没有 async 和 defer 属性—） 会阻断渲染并停止 HTML 的接卸；

（2）处理 CSS 文件，并构建 CSSOM 树

CSSOM 与 DOM 的数据模型非常相似， 它们是两颗树， 有独立的数据结构，浏览器将 CSS 规则解析成可以理解和使用的样式映射关系。浏览器遍历CSS中的每个规则集， 根据 CSS 选择器创建具有父、子和兄弟的的节点树；

（3）构建 Render 树

将 DOM 树 和 CSSOM 树组合成一颗 Render 树， 渲染树从DOM树的根节点开始构建， 遍历每个可见节点；

（4）布局 Paint

构建渲染树后，开始布局。渲染树标识显示哪些节点（即使不可见）及其计算样式，但不标识每个节点的尺寸或位置。为了确定每个对象的确切大小和位置，浏览器从渲染树的根开始遍历它。

第一次确定节点的大小和位置称为布局。随后对节点大小和位置的重新计算称为回流。假设初始布局发生在返回图像之前。由于我们没有声明图像的大小，因此一旦知道图像大小，就会有回流。

在渲染树上运行布局以计算每个节点的几何体。布局是确定呈现树中所有节点的宽度、高度和位置，以及确定页面上每个对象的大小和位置的过程。回流是对页面的任何部分或整个文档的任何后续大小和位置的确定。

（5）绘制内容

将各个节点绘制到屏幕上，第一次出现的节点称为first meaningful paint。在绘制或光栅化阶段，浏览器将在布局阶段计算的每个框转换为屏幕上的实际像素。绘画包括将元素的每个可视部分绘制到屏幕上，包括文本、颜色、边框、阴影和替换的元素（如按钮和图像）

合成
当文档的各个部分以不同的层绘制，相互重叠时，必须进行合成，以确保它们以正确的顺序绘制到屏幕上，并正确显示内容。

当页面继续加载资产时，可能会发生回流（回想一下我们迟到的示例图像），回流会触发重新绘制和重新组合。如果我们定义了图像的大小，就不需要重新绘制，只需要重新绘制需要重新绘制的层，并在必要时进行合成。但我们没有包括图像大小！从服务器获取图像后，渲染过程将返回到布局步骤并从那里重新开始。

5、浏览器与服务器断开请求；
